## Why SSH Tunnel

SSH Tunnel wraps the same `ssh -M` master-socket workflow you would run in
Terminal, then adds health checks, automatic reconnects, quick forwards,
diagnostics, and proper process lifecycle handling — so your tunnels stay in
sync without keeping Terminal windows open.

<CardGrid stagger>
  <Card title="Uses your SSH config" icon="setting">
    Reads tunnel hosts and `LocalForward` entries straight from `~/.ssh/config`
    via `ssh -G`. No separate config to maintain.
  </Card>
  <Card title="Self-healing tunnels" icon="random">
    Scheduled health checks plus TCP port probes, automatic reconnects with
    exponential backoff, and recovery on network changes and system wake.
  </Card>
  <Card title="Quick Forwards" icon="add-document">
    Forward a remote port on the fly. The app picks a free local port, saves
    it, and reapplies it after reconnects and restarts.
  </Card>
  <Card title="Native & lightweight" icon="apple">
    A SwiftUI menu bar app with no Dock icon. State at a glance through the
    menu bar icon color: idle, connecting, connected, or failed.
  </Card>
</CardGrid>
